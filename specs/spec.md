## 一、产品概述

### 1.1 产品愿景

记忆花园是一套基于 Web 的学习辅助系统，将抽象的学习过程转化为可视化的虚拟花园种植体验。用户每学习一条新知识，就在花园中种下一颗种子；通过持续的复习与巩固，植物从**种子 → 发芽 → 成长 → 开花 → 结果**逐步生长，直观反映知识的掌握程度。

### 1.2 核心价值主张

- **游戏化激励**：将枯燥的重复复习转化为有成就感的园艺体验
- **科学复习**：基于艾宾浩斯遗忘曲线，自动规划最佳复习时机
- **可视化反馈**：一眼看懂自己的知识掌握全景
- **习惯养成**：通过连续打卡、成就徽章等机制培养长期学习习惯

### 1.3 目标用户

- **全年龄段通用**：不限制特定人群
- **适用场景**：语言学习、考试备考、专业技能、兴趣知识等各类学习领域
- **使用频率**：轻量级到中强度，建议每日 5-30 分钟

## 三、核心功能详细说明

### 3.1 知识卡片

#### 数据结构

- **正面内容**：问题/提示（必填）
- **背面内容**：答案/解释（必填）
- **分类/标签**：归属哪个知识分类（必填，默认"未分类"）
- **备注**：可选补充信息
- **创建时间**：自动记录
- **最后复习时间**：自动更新

#### 卡片创建流程

1. 用户点击「新建知识」
2. 选择或新建分类
3. 填写正面内容和背面内容
4. （可选）填写备注
5. 提交 → 系统在花园中生成一颗**种子**

#### 预设知识库

- 系统提供若干预设知识包（如英语四级核心词汇、Java 基础概念等）
- 用户浏览并选择需要的知识包
- 选择后一键导入，批量生成种子

### 3.2 植物生长系统

#### 生长阶段定义

| 阶段 | 名称 | 视觉表现 | 触发条件 |
|------|------|----------|----------|
| Stage 1 | 种子 | 土壤中的小土包 | 知识刚创建 |
| Stage 2 | 发芽 | 小嫩芽冒出 | 首次复习且自评非"忘记" |
| Stage 3 | 成长 | 小树苗 | 第 2 次成功复习 |
| Stage 4 | 开花 | 植物开花 | 第 3 次成功复习 |
| Stage 5 | 结果 | 结出果实 | 连续多次成功复习，进入长期记忆 |

#### 生长规则

- **统一外观**：所有植物的视觉风格一致，仅通过生长阶段区分状态
- **正向反馈**：
  - 自评「记住了」→ 正常推进到下一阶段 / 维持当前阶段
  - 自评「模糊」→ 维持当前阶段，缩短下次复习间隔
  - 自评「忘记了」→ **回退一个阶段**，重新开始该阶段计数
- **枯萎机制**：
  - 超过当前阶段推荐复习时间的 **3 倍**未复习 → 进入枯萎状态
  - 枯萎表现：植物颜色变灰/变黄，叶片下垂
  - 复活方式：进行一次复习即可恢复到前一生长阶段

#### 艾宾浩斯复习间隔

```
第 1 次复习：学习后 1 天
第 2 次复习：第 1 次复习后 2 天
第 3 次复习：第 2 次复习后 4 天
第 4 次复习：第 3 次复习后 7 天
第 5 次复习：第 4 次复习后 15 天
第 6 次复习：第 5 次复习后 30 天
后续复习：每隔 30 天一轮
```

#### 错过复习的处理（智能重算）

- 如果用户在推荐日期之后才复习：
  - 计算实际经过的天数
  - 将实际天数代入曲线，推算出等效的复习轮次
  - 根据自评结果决定是推进/维持/回退
  - 重新计算下一次复习日期

### 3.3 复习流程

1. 用户登录系统，首页展示**今日待复习植物列表**
2. 用户点击某棵植物开始复习
3. 系统显示卡片的**正面内容**（问题）
4. 用户回忆答案后，点击「查看答案」
5. 系统显示**背面内容**（答案）
6. 用户选择自评结果：**记住了 / 模糊 / 忘记了**
7. 系统根据自评更新植物生长状态和下次复习时间
8. 自动跳转到下一棵待复习植物
9. 所有待复习完成后，展示**今日复习完成**总结页

### 3.4 花园视图

- **主入口**：登录后的默认页面
- **展示内容**：
  - 当前所有植物，按生长状态自动排列
  - 各阶段的植物分组显示（如"开花区""发芽区"等）
  - 待复习植物高亮标记
  - 枯萎植物特殊标识
- **筛选功能**：按分类筛选植物
- **排序选项**：按创建时间 / 按下次复习时间 / 按生长阶段

### 3.5 成就徽章系统

#### 徽章类型示例

| 徽章名称 | 解锁条件 | 稀有度 |
|----------|----------|--------|
| 初园丁 | 种植第 1 棵植物 | 普通 |
| 勤奋园丁 | 连续学习 7 天 | 普通 |
| 百花齐放 | 同时拥有 10 棵开花植物 | 稀有 |
| 硕果累累 | 首次收获果实（到达结果阶段） | 普通 |
| 知识森林 | 总共种植 100 棵植物 | 稀有 |
| 坚持不懈 | 连续学习 30 天 | 史诗 |
| 复活大师 | 成功复活 5 棵枯萎植物 | 普通 |
| 分类达人 | 创建超过 5 个知识分类 | 普通 |

#### 徽章展示

- 个人中心展示已获得的全部徽章
- 未获得的徽章以灰色显示，悬停提示解锁条件
- 获得新徽章时弹出庆祝动画

### 3.6 学习统计面板

#### 统计维度

- **今日数据**：
  - 新增知识数
  - 完成复习数
  - 植物生长/回退次数
- **本周 / 本月趋势图**：
  - 每日学习时长/次数折线图
  - 各阶段植物数量分布饼图
- **连续打卡**：
  - 当前连续天数
  - 历史最长连续天数
  - 日历热力图展示

## 五、非功能需求

### 5.1 可用性

- 系统需支持主流浏览器（Chrome、Firefox、Safari、Edge 最新两个版本）
- 页面加载时间不超过 3 秒
- 核心操作（提交复习）响应时间不超过 500ms

### 5.2 数据安全

- 密码加密存储（不可逆哈希）
- 用户数据隔离，不同用户间数据不可见（社区功能上线前）

### 5.3 可扩展性

- 知识卡片字段设计预留扩展空间
- 植物外观系统预留多皮肤接口
- 复习算法模块化，便于后续调整参数或替换为更复杂的模型

## 六、术语表

| 术语 | 定义 |
|------|------|
| 知识卡片 | 一条最小粒度的学习单元，包含问题和答案 |
| 植物 | 知识卡片在花园中的可视化表示 |
| 种子 | 植物的初始状态，对应刚创建的知识 |
| 生长阶段 | 种子→发芽→成长→开花→结果，共 5 个阶段 |
| 枯萎 | 因长期未复习而进入的退化状态 |
| 复活 | 通过复习将枯萎植物恢复正常状态的过程 |
| 艾宾浩斯遗忘曲线 | 一种描述人类记忆衰减规律的科学模型，用于计算最佳复习间隔 |
| 自评 | 用户每次复习后对自身掌握程度的主观评价（记住了/模糊/忘记了） |
| 徽章 | 达成特定条件后获得的成就奖励 |
| 打卡 | 当日完成至少一次复习操作 |

## 七、边缘场景与决策记录

### 7.1 已明确的决策

| # | 决策项 | 结论 | 理由 |
|---|--------|------|------|
| D-01 | 知识录入方式 | 支持手动创建 + 预设知识库 | 兼顾灵活性和易用性 |
| D-02 | 掌握程度判定 | 用户自评三档 | 简单直观，降低使用门槛 |
| D-03 | 枯萎机制 | 会枯萎但可复活 | 有适度压力但不打击积极性 |
| D-04 | 知识粒度 | 单条知识 = 一棵植物 | 最细粒度，便于精确追踪 |
| D-05 | 复习错过处理 | 按实际间隔智能重算 | 更符合真实使用情况 |
| D-06 | 植物外观 | 统一外观 | 降低设计和开发成本 |
| D-07 | 花园布局 | 自动排列 | 减少用户的整理负担 |
| D-08 | 提醒方式 | MVP 仅站内通知 | 最小化实现复杂度 |
| D-09 | 成就系统 | 仅徽章 | 聚焦核心激励机制 |
| D-10 | MVP 范围 | 核心功能优先 | 快速验证核心价值假设 |
